# A. Fill fields

```cpp
template <typename T> void register_reflected() {
    constexpr auto ctx = std::meta::access_context::current();
    ClassMeta &cls = Registry::instance().get_or_create(std::meta::identifier_of(^^T));

    template for (constexpr auto m: std::define_static_array(std::meta::nonstatic_data_members_of(^^T, ctx))) {
        cls.fields[std::string(std::meta::identifier_of(m))] = FieldMeta{
            .get = [](void* o) -> Any { return static_cast<T*>(o)->[: m :]; },
            .set = [](void* o, const Any& v) {
                static_cast<T*>(o)->[: m :] =
                std::any_cast<[: std::meta::type_of(m) :]>(v);
            },
        };
    }
}
```

This is a C++26 static reflection (P2996) function that auto-registers the fields of any type T into a runtime registry. Walking
through it:

## Setup

```cpp
constexpr auto ctx = std::meta::access_context::current();
```

Captures the access context at the call site. Reflection queries that take a context (like nonstatic_data_members_of) will then
honor the same private/protected visibility rules as if you wrote member-access code here. Call it from inside a friend or member
and you see private fields; call it from outside and you only see public ones.

```cpp
ClassMeta &cls = Registry::instance().get_or_create(std::meta::identifier_of(^^T));
```
- `^^T` is the reflection operator: it yields a `std::meta::info` — a `constexpr` handle to the entity `T`.
- `identifier_of(^^T)` returns T's unqualified name as a `string_view` (e.g. "Person").
- That name keys into a singleton Registry, returning a reference to the ClassMeta slot for this type.

## The expansion loop

```cpp
template for (constexpr auto m :std::define_static_array(std::meta::nonstatic_data_members_of(^^T, ctx))) {
```

- `nonstatic_data_members_of(^^T, ctx)` returns a `std::vector<std::meta::info>` of every non-static data member reflection of `T`,
filtered by ctx.
- `define_static_array(...)` promotes that vector into a static constexpr array with program-lifetime storage. This is required
because template for needs an iterable whose values are constant expressions; you can't iterate a `vector<info>` directly with it.
- `template for` is the C++26 expansion statement: the compiler unrolls the loop at compile time, producing one copy of the body
for each m. So if T has 3 fields, you get 3 inlined blocks — not a runtime loop.
 
## The body — one iteration per field
 
```cpp
cls.fields[std::string(std::meta::identifier_of(m))] = FieldMeta{
```

Stores a FieldMeta in the class's field map, keyed by the field name ("age", "name", …).
 
```cpp
.get = [](void* o) -> Any { return static_cast<T*>(o)->[: m :]; },
```

- `[: m :]` is the splicer: it converts the reflection m back into the language entity it refers to — here, a member identifier.
- So `static_cast<T*>(o)->[: m :]` expands at compile time into, e.g., `static_cast<T*>(o)->age`.
- The lambda erases T behind void* so the registry can store get/set callbacks uniformly across all types.

```cpp 
.set = [](void* o, const Any& v) {static_cast<T*>(o)->[: m :] = std::any_cast<[: std::meta::type_of(m) :]>(v);},
```

- `type_of(m)` is the reflection of the member's declared type.
- `[: std::meta::type_of(m) :]` splices that type into the template argument of any_cast. So for an int age field, this expands to 
std::any_cast<int>(v).
- The result is assigned back through the spliced member.

## Importance

Each iteration of template for produces a different specialization of the lambdas — m and its type are baked in as constants. 
There is no runtime reflection cost on get/set; the boilerplate of “for each field, write a get and set wrapper” is generated by
the compiler from the type alone. Add a field to T and the registration code follows automatically the next time the translation 
unit is compiled.

# B. Fill methods

```cpp
template <typename T, std::meta::info Fn, std::size_t... Is>
auto make_invoker(std::index_sequence<Is...>) {
    return [](void *obj, std::span<const Any> args) -> Any {
        constexpr auto params =
            std::define_static_array(std::meta::parameters_of(Fn));
        return Any{(static_cast<T *>(obj)->[: Fn :])(
            std::any_cast<std::remove_cvref_t<
                typename[: std::meta::type_of(params[Is]) :]>>(args[Is])...)};
    };
}
```

## What it does

`make_invoker` is the method counterpart to the field get/set lambdas above. Given a class `T` and the reflection of one of its
member functions `Fn`, it returns a single, type-erased callable

```cpp
Any (void* obj, std::span<const Any> args)
```

that, when invoked, unpacks `args` into the function's real parameter types and calls `obj->Fn(...)`. Every registered method ends
up behind this same signature, so a method table can hold them all in one container.

## How the arguments find their types

The whole problem `make_invoker` solves is bridging two worlds:

- The **caller** has `std::span<const Any>` — values whose static types are erased.
- The **callee** has a concrete signature like `void greet(const std::string&, int)` — each parameter needs its real type back.

The bridge is built from three pieces working together: a reflection pack, an index pack, and a splicer for types.

### 1. The reflection of `Fn` as a template parameter

```cpp
template <typename T, std::meta::info Fn, std::size_t... Is>
```

`Fn` is declared as a non-type template parameter of type `std::meta::info` — a C++26 capability. Because `Fn` is a template
parameter, it is a constant expression inside the function body, which is exactly what the splicer `[: Fn :]` requires.

### 2. The index pack as a "for loop over parameters"

```cpp
auto make_invoker(std::index_sequence<Is...>) { ... }
```

C++ has no syntax to write "for each parameter index, do X" inside a single expression. The classical workaround is to take an
`std::index_sequence<0, 1, …, N-1>` as a function argument so its indices land in scope as a parameter pack `Is...`. Anything
written with `... Is ...)...` inside the lambda then expands once per parameter position.

The caller side looks like:

```cpp
make_invoker<T, ^^&T::greet>(
    std::make_index_sequence<std::meta::parameters_of(^^&T::greet).size()>{});
```

### 3. The pack expansion that does the real work

```cpp
constexpr auto params =
    std::define_static_array(std::meta::parameters_of(Fn));

return Any{(static_cast<T*>(obj)->[: Fn :])(
    std::any_cast<std::remove_cvref_t<
        typename [: std::meta::type_of(params[Is]) :]>>(args[Is])...)};
```

`parameters_of(Fn)` is a `vector<std::meta::info>` of the function's declared parameters; `define_static_array` lifts it into a
program-lifetime `constexpr` array so `params[Is]` is itself a constant expression — required, because the next step splices it.

Read the body inside-out, one parameter index at a time:

| piece | what it produces for parameter *i* |
| --- | --- |
| `params[Is]` | reflection of parameter *i* |
| `std::meta::type_of(params[Is])` | reflection of that parameter's **type** |
| `typename [: … :]` | splices that type back into the language as a real type |
| `std::remove_cvref_t<…>` | strips `const` / `&`, since `Any` stores by value |
| `std::any_cast<…>(args[Is])` | pulls the *i*-th argument out of the span at that concrete type |

The trailing `...` then expands the whole sub-expression over the pack. For a method `void f(const std::string&, int)` the
expansion is, after splicing:

```cpp
static_cast<T*>(obj)->f(
    std::any_cast<std::string>(args[0]),
    std::any_cast<int>(args[1]))
```

Finally `static_cast<T*>(obj)->[: Fn :]` splices `Fn` into a member-access — yielding the actual member-function reference — and
the outer `Any{ ... }` wraps the return value back into the erased representation.

## Why this matters

- **No virtual dispatch, no member-pointer indirection.** Because `Fn` is a compile-time reflection, `[: Fn :]` resolves to a
  direct call to that exact member function. The compiler typically inlines it.
- **Per-method specialization.** Every registered method gets its own instantiation of the lambda with the right types baked in.
  Adding or changing a method's signature regenerates the unpacking automatically.
- **One uniform signature at the boundary.** All the reflection work happens inside; outside, callers see a plain
  `(void*, span<Any>) -> Any` — easy to store, pass around, or expose to a scripting layer.

Caveat: the body wraps the call in `Any{ ... }`, so this form assumes a non-`void` return. A `void`-returning variant would need
either a separate overload or an `if constexpr (std::is_void_v<…>)` branch around the return.
